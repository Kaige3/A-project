# 产品开发文档 (Product Development Documentation)

这是一份关于产品开发、AI 辅助编程（AI 结对编程）以及常用开发工具的文档沉淀库。旨在帮助开发者在使用 AI 辅助开发时，能够更高效地进行架构决策、需求拆解和代码落地，同时提供基础工具的速查指南。

## 📂 目录结构与内容说明

### 1. Prompt (AI 提示词与决策模板)
存放与 AI 交互时的高效 Prompt 模板，涵盖从思路雏形到工程化落地的全场景：
- **`AI决策模板Prompt.md`**：提供 4 个常用模板（初期思路纠偏、多方案对比、一键输出 PRD+SOP、头脑风暴与风险兜底），帮助开发者利用 AI 梳理思路、排雷并输出标准文档。
- **`程序架构Prompt.md`**：面向程序员的专属模板，重点在于架构纠偏、逻辑压测、技术方案横向对比（SWOT+RICE）以及工程化落地（接口定义与验收标准）。

### 2. SOP流程 (标准作业程序)
存放开发过程中的标准方法论与流程规范：
- **`vibe_coding_sop.md`**：核心总结了“AI 结对编程的五步法则”（高层建瓴选架构、胖后端瘦前端解耦、真实数据结构对齐、防御性漏洞扫描、多模态视觉纠偏），明确了开发者作为“架构师”，AI 作为“执行者”的最佳协作模式。

### 3. 工具总结扫盲 (基础工具指南)
存放日常开发必备的工具使用速查手册：
- **`git常用指令大全.md`**：详细分类整理了 Git 的高频指令，包括本地配置、基础提交、分支管理、远程同步和版本回退，适合开发者快速查阅和直接复制使用。

## 🚀 如何使用
- **需求探索与架构选型**：在开发初期，复制 `Prompt/` 目录下的模板与 AI 进行深度对话，完成技术选型、风险排查和架构设计。
- **AI 辅助编码执行**：在实际写代码阶段，严格遵循 `sop流程/vibe_coding_sop.md` 中的五步法则，确保 AI 生成代码的准确性和系统可维护性。
- **日常工具速查**：遇到 Git 操作疑问或遗忘时，随时查阅 `工具总结扫盲/` 目录。
